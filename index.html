Big X Reality · Multiplayer VR / PC-VR

Marine Corps VR Training Simulator

A large-scale VR training simulation platform combining Windows PC clients, PC-to-VR streamed clients, motion platforms, external tracking systems, weapon peripherals, NPC systems and backend data workflows.

My Role

Senior Unreal Engineer responsible for Unreal-side systems, NPC behavior design and implementation, reusable tools, data collection workflows, runtime tuning support, multiplayer test workflows and hardware-integration support.

Technical Scope

  • Mixed desktop and PC-to-VR streamed training workflows
  • Multiplayer simulation support for up to 30 connected computers
  • NPC behavior design and implementation for training scenarios
  • Performance and network bottleneck investigation with Unreal Insights
  • Motion platform / I/O signal integration through shared-memory workflows
  • Antilatency, OptiTrack and weapon peripheral integration support
  • HTTP / Socket.IO data collection and backend handoff workflows

Engineering Challenge

The project required multiple user roles, VR clients, desktop clients, NPCs and hardware systems to be validated in packaged builds. I used Unreal Insights to trace performance and network transmission bottlenecks, then adjusted Unreal-side systems and workflows to improve stability and iteration quality.

Impact

  • Supported large-scale multiplayer validation with up to 30 connected computers.
  • Built NPC behavior systems that could be reused across training scenarios.
  • Used profiling evidence from Unreal Insights to identify and fix performance and networking bottlenecks.
  • Reduced repetitive setup by creating packaged-build launch workflows for server and selected client types.
  • Enabled non-Unreal teammates to tune packaged EXE parameters through external Excel/config files.
